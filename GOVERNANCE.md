# Class Q Governance

Class Q is stewarded by the Class Q Foundation, a vendor‑neutral, non‑profit organization dedicated to maintaining the open standard.

---

## Principles

- Openness  
- Technical rigor  
- Interoperability  
- Longevity  
- Artist‑centric design  
- Vendor neutrality  

---

## Structure

### Steering Committee
Responsible for:
- approving spec versions  
- resolving cross‑group conflicts  
- long‑term roadmap  

### Working Groups
Each subsystem has a dedicated Working Group:
- Q‑DSD  
- Q‑Link  
- Q‑Core  
- Q‑DSP  
- Q‑Time  
- Q‑Stream  
- Q‑Disc  
- Q‑Amp  
- Q‑IEM  
- Q‑Venue  
- Q‑Studio  
- Q‑DRM  

### Contributors
Anyone may propose changes, submit PRs, or join discussions.

---

## Decision Process

1. Proposal submitted via GitHub Issue  
2. Discussion within relevant Working Group  
3. Draft revision  
4. Consensus vote  
5. Steering Committee approval  
6. Versioned release  

---

## Versioning

Class Q uses semantic versioning for specifications:
- **MAJOR** — breaking changes  
- **MINOR** — new features  
- **PATCH** — clarifications or corrections  

---

## Amendments

Governance rules may be updated by Steering Committee vote.
