# Contributing to Class Q

Thank you for your interest in contributing to the Class Q Open Standard.

Class Q is a vendor‑neutral, royalty‑free, open standard intended to last for generations.  
We welcome contributions from engineers, researchers, manufacturers, studios, venues, and developers.

---

## How to Contribute

### 1. Propose changes
Open a GitHub Issue describing:
- the change you propose  
- the rationale  
- any technical references  
- potential impact on interoperability  

### 2. Submit Pull Requests
Pull Requests should:
- be scoped and focused  
- include clear explanations  
- reference related Issues  
- avoid proprietary or confidential information  

### 3. Join a Working Group
Each subsystem (Q‑DSD, Q‑Link, Q‑Core, etc.) has a dedicated Working Group.  
Participation is open to all contributors.

---

## Contribution Requirements

- All contributions must be original work.  
- No patented material unless explicitly licensed for open use.  
- No reverse‑engineered or proprietary content.  
- All contributions fall under the Apache 2.0 license.

---

## Code Style (Reference Implementations)

Reference code should be:
- minimal  
- educational  
- implementation‑agnostic  
- free of hardware‑specific assumptions  

---

## Reporting Issues

Please include:
- clear reproduction steps  
- expected vs. actual behavior  
- environment details  
- relevant logs or traces  

---

## Community

Class Q is a collaborative effort.  
We encourage respectful, constructive discussion and technical rigor.

See **CODE_OF_CONDUCT.md** for community guidelines.
